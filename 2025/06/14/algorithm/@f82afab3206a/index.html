# 【C++ 筆記】基礎排序（氣泡、選擇、插入排序法）

本筆記僅供個人學習用途，內容斟酌參考。

# 1.Bubble Sort

原理：兩兩相鄰比較，大的往後推，重複進行多輪直到完成排序。

時間複雜度：

  • 最佳情況： O(n)O(n)O(n) （若資料已排序，可加入 flag 判斷是否有交換，避免多餘比較）
  • 平均情況： O(n2)O(n^2)O(n2) （每個元素都要比較一次）
  • 最壞情況： O(n2)O(n^2)O(n2) （完全反序，每輪皆需交換到最後）

空間複雜度： O(1)O(1)O(1)

算是穩定的排序法，也最常用。

# 2.Selection Sort

原理：每輪選出剩下的最小值，放到目前排序的位置。

時間複雜度：

  • 最佳情況： O(n2)O(n^2)O(n2) （無法透過資料特性提前結束，仍需全部比較）
  • 平均情況： O(n2)O(n^2)O(n2) （一共進行 n(n−1)2\frac{n(n-1)}{2}2n(n−1)​ 次比較）
  • 最壞情況： O(n2)O(n^2)O(n2)

空間複雜度： O(1)O(1)O(1)

該排序法不是那麼穩，因為交換過程中有可能會打亂相同元素的順序。

# 3.Insertion Sort

原理：從左至右逐一「插入」目前元素到左邊已排序好的序列中。

時間複雜度：

  • 最佳情況： O(n)O(n)O(n) （若原序列已排序，不需移動，只需一次比較）
  • 平均情況： O(n2)O(n^2)O(n2) （中間情況需要約一半元素移動）
  • 最壞情況： O(n2)O(n^2)O(n2) （若為反序，則每插入一個元素都要移動全部前面的元素）

空間複雜度： O(1)O(1)O(1)

與 Bubble Sort 一樣穩，原因是不會改變相同值的順序。

# 總結

排序法 最佳時間複雜度 平均時間複雜度 最壞時間複雜度 空間複雜度 穩定性
Bubble Sort O(n) O(n²) O(n²) O(1) 穩
Selection Sort O(n²) O(n²) O(n²) O(1) 不穩
Insertion Sort O(n) O(n²) O(n²) O(1) 穩
