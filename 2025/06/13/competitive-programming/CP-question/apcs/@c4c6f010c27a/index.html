# 【APCS】2023 年 1 月實作題 C++ 解題筆記（前兩題）

此筆記僅供個人學習用途，內容僅供參考。

  1. https://zerojudge.tw/ShowProblem?problemid=j605

題目說明：

有 KKK 個提交紀錄，第 iii 個提交紀錄有兩整數 ti,sit_i, s_iti​,si​ ，分別為上傳時間跟提交分數，若第 iii 次的提交結果為嚴重錯誤，則 si=−1s_i = -1si​=−1 。

計算總分公式：提交紀錄最高分 −-− 總提交次數 −-− 總嚴重次數 ×2\times 2×2。

若總分算出來 <0< 0<0 ，則計為 000 。

輸出一行，總分和第一次最高分的時間點。

解題思路：

  1. 宣告兩變數 ans, sum_s ，分別用來計算總分跟總嚴重錯誤次數
  2. 輸入的時候就可以先判斷是否有 s[i] == -1 的情形。
  3. max_element(s.begin(), s.end()); 計算值最大的元素，但記得這回傳結果會是迭代器，要加上 * 解參考運算子（Dereference）。
  4. 用 find(s.begin(), s.end(), max_score); 跟 distance 去算第一次出現最高分的 index。
  5. 最後就套題目給的公式去解，還有固定的輸出格式。
  1. https://zerojudge.tw/ShowProblem?problemid=j606

題目說明：

有個初始字串 SSS ，長度為 KKK ，每字元皆為小寫的英文字母。

接下來 QQQ 次修改，會給 QQQ 行 1∼K1 \sim K1∼K 數字，這些數字可以使 SSS 重新排列，這些數字都是以 111 為起始點的 index。

如： "abac" ， P=[4,1,3,2]P = [4, 1, 3, 2]P=[4,1,3,2] ，可得到新字串 "bcaa" 。

輸出這樣操作完後字串的第 1∼R1 \sim R1∼R 個字元，並輸出 RRR 行。

如範例輸入 1：

5 4 1
abcde
2 1 3 5 4
5 1 2 4 3
4 1 2 3 5
3 1 4 5 2

範例輸出 1：

bacd

bacd 是從這四次的新字串中，都取每個新字串的第一個字元出來的。

解題思路：

  1. 用 C++ 內建 string ，以 getline 輸入，記得前面要加上 cin.ignore() ，因為前面有 K, Q, R 的 cin 輸入。
  2. vector <string> pS(Q) 表示每次做排列操作的字串，有 QQQ 個。
  3. 雙層 for 迴圈中，外層做一個 temp string 讓他先 =S= S=S 。
    • 再設一個 array 叫 int P[K]; ，也是一個 temp，用來暫時存數字排列的陣列。
    • 內層迴圈輸入 P[j]; ，並同時做排列操作 new_S[P[j] - 1] = S[j]; 。記得 P[j]-1 因為題目是以 1 為起始的索引。
    • 結束後又回到外層迴圈，更新 S = new_S ，並將操作完的字串加入 pS 。
