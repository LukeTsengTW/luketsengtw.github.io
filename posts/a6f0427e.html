【C++】競程筆記（多維 DP、LCS 最長共同子序列）

題目範例參考：NTUCPC Guide，此筆記僅為個人學習用途。

什麼是多維的 DP?

一開始入門 DP 所寫的全都是一維的 DP，例如在爬樓梯問題， dp[i]dp[i]dp[i] 就表示成到達第 iii 階的方法數。

而當中只有一個變數，叫做 iii ，故稱為一維 DP。

多維的 DP 就是有多個變數的 DP，一個變數不足以清楚描述當前的子問題，需要兩個或更多的變數來鎖定狀態。

例如：

  • 走迷宮需要 x,yx, yx,y 座標兩個變數來表示 dp[x][y]dp[x][y]dp[x][y]
  • LCS（Longest Common Subsequence：最長共同子序列）需要字串 A 的位置 iii 和字串 B 的位置 jjj 表示 dp[i][j]dp[i][j]dp[i][j]
  • 背包問題：需要物品編號 iii 和剩餘重量 www 表示 dp[i][w]dp[i][w]dp[i][w]

Vacation

Problem Source：https://atcoder.jp/contests/dp/tasks/dp_c

狀態定義：不能只紀錄 dp[i]dp[i]dp[i] 代表第 iii 天的最大快樂值，因為要知道第 iii 天到底做了什麼活動，才能決定第 i+1i+1i+1 天不能做什麼。

這邊可以應用多維 DP 的想法，擠在同一個 DP 裡面：

  • dp[i][0]dp[i][0]dp[i][0] ：第 iii 天選擇活動 A 的最大總快樂值。
  • dp[i][1]dp[i][1]dp[i][1] ：第 iii 天選擇活動 B 的最大總快樂值。
  • dp[i][2]dp[i][2]dp[i][2] ：第 iii 天選擇活動 C 的最大總快樂值。

但也可以分成三個 dp 去宣告做狀態定義：

  • dpa[i]dpa[i]dpa[i]
  • dpb[i]dpb[i]dpb[i]
  • dpc[i]dpc[i]dpc[i]

用哪一個都沒差，最後寫法還是一樣。

轉移式定義：

  • dp[i][0]=a[i]+max⁡(dp[i−1][1],dp[i−1][2])dp[i][0] = a[i] + \max(dp[i-1][1], dp[i-1][2])dp[i][0]=a[i]+max(dp[i−1][1],dp[i−1][2])
  • dp[i][1]=b[i]+max⁡(dp[i−1][0],dp[i−1][2])dp[i][1] = b[i] + \max(dp[i-1][0], dp[i-1][2])dp[i][1]=b[i]+max(dp[i−1][0],dp[i−1][2])
  • dp[i][2]=c[i]+max⁡(dp[i−1][0],dp[i−1][1])dp[i][2] = c[i] + \max(dp[i-1][0], dp[i-1][1])dp[i][2]=c[i]+max(dp[i−1][0],dp[i−1][1])

如果今天選 A，那昨天一定只能選 B 或 C（取昨天兩者中較大的一個）。

如果今天選 B，那昨天一定只能選 A 或 C。

如果今天選 C，那昨天一定只能選 A 或 B。

題目限制兩天不能做一樣的事情。

base case 初始狀態：

  • dp[0][0]=a[0]dp[0][0] = a[0]dp[0][0]=a[0]
  • dp[0][1]=b[0]dp[0][1] = b[0]dp[0][1]=b[0]
  • dp[0][2]=c[0]dp[0][2] = c[0]dp[0][2]=c[0]

股票買賣 V

Problem Source：https://www.luogu.com.cn/problem/U425829

先前的做法是先定義好狀態跟轉移式，之後再透過整理轉移式得到 O(n)O(n)O(n) 解。

但學會多維 DP 後，可直接得到 O(n)O(n)O(n) 解，省去整理轉移式的麻煩。

狀態定義：

  • dp[i][0]dp[i][0]dp[i][0] ：有股票。可能是今天剛買的，或者之前買了還沒賣。
  • dp[i][1]dp[i][1]dp[i][1] ：剛賣出股票。今天賣出，導致明天進入冷凍期。
  • dp[i][2]dp[i][2]dp[i][2] ：沒股票且非冷凍期。可能是昨天就沒股票，或者昨天是冷凍期今天解凍了，是隨時可以買入的狀態。

轉移式定義：

  • 持有股票的狀態， dp[i][0]dp[i][0]dp[i][0]
    • 狀況 A：昨天就持有股票，今天不賣 →dp[i−1][0]\rightarrow dp[i-1][0]→dp[i−1][0]
    • 狀況 B：昨天沒股票也沒被冷凍，今天買入 →dp[i−1][2]−prices[i]\rightarrow dp[i-1][2] - prices[i]→dp[i−1][2]−prices[i]

dp[i][0]=max⁡(dp[i−1][0],dp[i−1][2]−prices[i])dp[i][0] = \max(dp[i-1][0], dp[i-1][2] - prices[i])dp[i][0]=max(dp[i−1][0],dp[i−1][2]−prices[i])

  • 剛賣出股票 dp[i][1]dp[i][1]dp[i][1]
    • 昨天持有股票，今天賣掉。

dp[i][1]=dp[i−1][0]+prices[i]dp[i][1] = dp[i-1][0] + prices[i]dp[i][1]=dp[i−1][0]+prices[i]

  • 沒股票且非冷凍期 dp[i][2]dp[i][2]dp[i][2]：
    • 狀況 A：昨天沒買股票，繼續觀望 →dp[i−1][2]\rightarrow dp[i-1][2]→dp[i−1][2]
    • 狀況 B：昨天剛賣出，今天強制休息（冷凍期） →dp[i−1][1]\rightarrow dp[i-1][1]→dp[i−1][1]

dp[i][2]=max⁡(dp[i−1][2],dp[i−1][1])dp[i][2] = \max(dp[i-1][2], dp[i-1][1])dp[i][2]=max(dp[i−1][2],dp[i−1][1])

初始狀態定義，第 0 天（第一筆資料）：

  • dp[0][0]dp[0][0]dp[0][0]（持有）：只能是當天買入，利潤為負 →−prices[0]\rightarrow -prices[0]→−prices[0]
  • dp[0][1]dp[0][1]dp[0][1]（剛賣）：第一天不可能賣出，設為極小值或 0（邏輯上不可能發生） →0\rightarrow 0→0
  • dp[0][2]dp[0][2]dp[0][2]（沒股票、冷凍期）：第一天什麼都不做 →0\rightarrow 0→0

組合數

Problem Source：https://www.luogu.com.cn/problem/U159710

這題是關於組合數 C(n,m)C(n, m)C(n,m) 的計算。

組合公式：$$C(n, m) = \frac{n!}{m!(n-m)!}$$

計算階乘是非常慢的，而且題目的查詢量 qqq 有到 10410^4104 ，因此改用二維 DP 結合帕斯卡三角形來做： $$C^n_m = C^{n-1}m + C^{n-1}{m-1}$$

狀態定義：定義 dp[i][j]dp[i][j]dp[i][j] 為從 iii 個元素中選出 jjj 個的方法數，也就是 C(i,j)C(i, j)C(i,j)。

轉移式定義：

根據帕斯卡三角形的性質「從 iii 個元素選 jjj 個」的方法數等於以下兩者之和：

  • 不包含第 iii 個元素：前 i−1i-1i−1 個裡面選 jjj 個 →dp[i−1][j]\rightarrow dp[i-1][j]→dp[i−1][j]
  • 包含第 iii 個元素：前 i−1i-1i−1 個裡面選 j−1j-1j−1 個 →dp[i−1][j−1]\rightarrow dp[i-1][j-1]→dp[i−1][j−1]

因此得到完整的轉移式：$$dp[i][j] = (dp[i-1][j-1] + dp[i-1][j]) \pmod{10^9 + 7}$$

初始狀態定義：選 0 個，從任意 iii 個元素中選 0 個，只有一種方法（都不選）。$$dp[i][0] = 1$$

由於是有多筆查詢，而且 N 數量較少，所以改成直接建表，之後每筆查詢都能用 O(1)O(1)O(1) 時間做到。

最長共同子序列（Longest Common Subsequence, LCS）

LCS 是非常經典且重要的演算法問題，特別在字串處理、生物資訊學（DNA 比對）以及版本控制系統（Git 的 diff 功能）等中都有廣泛應用。

那這也是很經典的 DP 例題，非學不可。

子字串 vs. 子序列

  • 子字串（Substring）：必須是連續的。
    • 例如：“ABCDE” 的子字串是 “BCD”。
  • 子序列（Subsequence）：不一定要連續，但先後順序不能改變。
    • 例如：“ABCDE” 的子序列可以是 “ACE” 或 “BD”。

LCS 定義

給定兩個序列（字串）S1S1S1 和 S2S2S2，找出一個最長的序列，同時出現在 S1S1S1 和 S2S2S2 中。

假設有兩個字串 A、B：

  • 字串 A："ABCBDAB"
  • 字串 B："BDCABA"

它們的 LCS 是 "BCBA"。

LCS 可能不只一個，如 "BDAB" 也是長度為 4 的共同子序列，但通常求的都是「最長的長度」。

818 . Longest Common Subsequence

Problem Source：https://oj.ntucpc.org/problems/818

  1. 狀態定義： dp[i][j]dp[i][j]dp[i][j] 代表「字串 A 的前 iii 個字元」與「字串 B 的前 jjj 個字元」 的 LCS 長度。

當 i=0i=0i=0 時，代表字串 A 取了 0 個字元（空字串）。

當 i=n,j=mi=n, j=mi=n,j=m 時， dp[n][m]dp[n][m]dp[n][m] 即為兩完整字串的 LCS 長度。

  1. 轉移式定義
    • 字元相同：如果 AAA 的第 iii 個字元等於 BBB 的第 jjj 個字元（A[i−1]=B[j−1]A[i-1] = B[j-1]A[i−1]=B[j−1]），該字元必屬於 LCS 的一部分。 $$dp[i][j] = dp[i-1][j-1] + 1$$
    • 字元不同： A[i−1]≠B[j−1]A[i-1] \neq B[j-1]A[i−1]=B[j−1] ，兩字元無法同時成為 LCS 的結尾。 $$dp[i][j] = \max(dp[i-1][j], \quad dp[i][j-1])$$

至於字元不同的情況，那 A[i]A[i]A[i] 和 B[j]B[j]B[j] 不可能同時被選入當作結尾，此時有兩種選擇：

  • A[i]A[i]A[i] 沒用，把它丟掉 →\rightarrow→ 問題變成找 「AAA 的前 i−1i-1i−1 個」跟「BBB 的前 jjj 個」。
  • B[j]B[j]B[j] 沒用，把它丟掉 →\rightarrow→ 問題變成找 「AAA 的前 iii 個」跟「BBB 的前 j−1j-1j−1 個」。

在這兩種選擇中取兩者中的最大值，因為要求 LCS。

  1. 初始狀態定義 $$dp[0][j] = 0, \quad \text{for } 0 \le j \le m$$ $$dp[i][0] = 0, \quad \text{for } 0 \le i \le n$$
