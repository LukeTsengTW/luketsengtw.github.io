4. Uva 10365 - Blocks

PDF Source：https://onlinejudge.org/external/103/10365.pdf

Uva Online Judge：https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=1306

Zerojudge：https://zerojudge.tw/ShowProblem?problemid=r579

難度：★★★☆☆

題目翻譯：

Donald 想送一份禮物給他的新侄子 Fooey。Donald 是個有點傳統的人，所以他選擇送一組由 NNN 個經典嬰兒積木組成的禮物。每個積木都是一個 111 英寸乘以 111 英寸乘以 111 英寸的正方體。Donald 想把這些積木堆疊成一個長方體，並用牛皮紙包裝起來以便寄送。請問 Donald 需要多少牛皮紙？

輸入的第一行包含一個整數 CCC，代表測資的數量。對於每個測資，會額外提供一行包含一個整數 NNN，即要運送的積木數量。NNN 不會超過 1000。

你的程式應針對每個測資產生一行輸出，給出將積木堆疊在一起時，包裝所需的最少紙張面積（單位為平方英寸）。

Sample Output:

題目說明:

現在有 NNN 個大小完全一樣的積木（每個都是 1x1x1 的小正方體），要把這 NNN 個積木全部用上，拼成一個實心的、沒有空隙的大長方體，最後用牛皮紙把整個大長方體包裝起來。

題目問的是要找出哪一種拼法可以最省包裝紙，並算出最少需要多少面積的紙。

從題目給出的資訊，可以得出以下這些資訊：

  • 積木總數 NNN 就是長方體的體積。
  • 包裝紙的用量就是長方體的表面積。
  • 要找出三個正整數來代表長方體的長（LLL）、寬（WWW）、高（HHH）。

限制條件：L×W×H=NL \times W \times H = NL×W×H=N （確保剛好用完 NNN 個積木）。

求解公式：在滿足上述條件下，讓表面積公式 2(L×W+W×H+H×L)2(L \times W + W \times H + H \times L)2(L×W+W×H+H×L) 的計算結果達到最小值。

解題思路:

這題即為窮舉法，因為 NNN 最高只到 100010001000，另外有些地方是可以做剪枝的，如下：

  • 避免重複計算相同的組合（例如 1×2×51 \times 2 \times 51×2×5 和 2×1×52 \times 1 \times 52×1×5），可強制規定邊長的大小關係為 L≤W≤HL \le W \le HL≤W≤H。
  • 基於上述規定，最小的邊 LLL 絕對不可能大於 NNN 的立方根，所以在第一層迴圈中，條件設為 l * l * l <= n 即可。
  • 當確定 LLL 後，剩下的兩邊乘積為 N/LN / LN/L （ W×H=N/LW \times H = N / LW×H=N/L ），第二小的邊 WWW 必須 >= LLL，且絕對不可能大於 N/LN / LN/L 的平方根，所以第二層迴圈條件為 w * w <= remain， remain 是剩下的面積 N/LN / LN/L。

接著在雙重迴圈中，透過取餘數 % 來檢查當前的 LLL 和 WWW 是否能整除總數，如果可以，就能算出第三個邊 HHH，然後套用表面積公式，用 min() 去做比較跟更新。

:::info
至於最小的邊 LLL 絕對不可能大於 N3\sqrt[3]{N}3N​ ，這個是因為前面已經限制 L≤W≤HL \le W \le HL≤W≤H ，假設 W=H=LW = H = LW=H=L ，則 L3≤N→L≤N3L^3 \le N \rightarrow L \le \sqrt[3]{N}L3≤N→L≤3N​ ，因此得到最小的邊 LLL 絕不可能大於 N3\sqrt[3]{N}3N​，而第二小的邊 WWW 絕對不可能大於 N/LN / LN/L 的平方根的原理也是同理。
:::

5. Uva 11039 - Building designing

PDF Source：https://onlinejudge.org/external/110/11039.pdf

Uva Online Judge：https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=1980

Zerojudge：https://zerojudge.tw/ShowProblem?problemid=d731

難度：★★☆☆☆

題目翻譯：

一位建築師想要設計一棟非常高的建築。這棟建築由若干樓層組成，且每一層樓都有特定的尺寸。每一層樓的尺寸必須大於其正上方那一層樓的尺寸。此外，設計師（他是某支著名西班牙足球隊的粉絲）想要將建築塗成藍色和紅色，每一層塗一種顏色，且規定相鄰兩個樓層的顏色必須不同。

為了設計這棟建築，建築師擁有 nnn 個可用的樓層，並已知它們各自的尺寸與顏色。所有可用樓層的尺寸皆不相同。建築師希望在滿足上述限制的情況下，利用這些可用樓層設計出最高（樓層數最多）的建築。

Input:

輸入檔案的第一行包含一個整數 ppp，代表需要求解的測資數量。每個測資的第一行包含可用樓層的數量。接著，每一行會出現一個樓層的尺寸與顏色，由一個介於 −999999-999999−999999 到 999999999999999999 之間的整數表示（不包含 0）。

  • 負數代表紅色樓層。
  • 正數代表藍色樓層。
  • 樓層的尺寸即為該數字的絕對值。
  • 不存在兩個尺寸相同的樓層。
  • 單一問題的最大樓層數為 500,000500,000500,000。

Output:

針對每個測資，輸出應包含一行文字，顯示在上述條件下所能建出的最高建築之樓層總數。

Sample Input:

Sample Output:

解題思路:

這題就是貪婪+排序。

要蓋出最高的建築，目標就是盡可能選多一點的樓層，根據題目限制，樓層必須滿足兩個條件：

  1. 尺寸限制：由下到上，樓層的絕對值要嚴格遞減（底層最大，頂層最小）。
  2. 顏色限制：相鄰樓層的顏色必須不同（即正負號交替）。

在尺寸限制上，可以先做排序，但會有個問題就是顏色問題（有正有負），這部分在排序函式裡面加個絕對值即可。

顏色限制上，可以透過兩個變數來做到：int color = (floor[i] > 0) ? 1 : -1; 表示現在的顏色，int lastColor = 0; 表示最後的顏色是什麼。

如果 lastColor == 0 表示還沒有任何顏色，樓高可以 ++，並更新 lastColor；若下一輪 lastColor != color，就表示現在的顏色跟上一個顏色不同。

6. Uva 11526 - H(n)

Zerojudge：https://zerojudge.tw/ShowProblem?problemid=d193

難度：★★★★★

題目翻譯：

以下 C++ 函式會回傳出什麼值？

Input:

第一行輸入為整數 TTT（T≤1000T \le 1000T≤1000），表示測資數量，接下來的每 TTT 行都會包含一個有號的 32 位元整數 nnn。

Output:

對於每個測資，輸出只會有一行包含 H(n)H(n)H(n)。

Sample Input:

Sample Output:

解題思路:

題目的時限是 5 秒，而且 nnn 最大可到 20 億，測資又有 1000 筆，如果照寫程式碼的話，時間複雜度會是 O(n)O(n)O(n)，絕對會超時，因此要想辦法降到 O(n)O(\sqrt{n})O(n​)。

在此會用到的一種技巧叫做 Square Root Decomposition，稱之為分塊。

來觀察一下 n/in/in/i 在這段函式裡的的變化。假設 n=10n = 10n=10：

  • i=1i = 1i=1：10/1=1010 / 1 = 1010/1=10
  • i=2i = 2i=2：10/2=510 / 2 = 510/2=5
  • i=3i = 3i=3：10/3=310 / 3 = 310/3=3
  • i=4i = 4i=4：10/4=210 / 4 = 210/4=2
  • i=5i = 5i=5：10/5=210 / 5 = 210/5=2
  • i=6i = 6i=6：10/6=110 / 6 = 110/6=1
  • i=7i = 7i=7：10/7=110 / 7 = 110/7=1
  • i=8i = 8i=8：10/8=110 / 8 = 110/8=1
  • i=9i = 9i=9：10/9=110 / 9 = 110/9=1
  • i=10i = 10i=10：10/10=110 / 10 = 110/10=1

會發現在這裡面，隨著 iii 變大，n/in/in/i 的結果會出現大量重複的區塊，像是當 iii 從 6 到 10 時，n/in/in/i 的值都是 1。

既然值都一樣，就不用一個一個慢慢加，而是可以直接算出這個區塊的長度，然後用乘法一次加總起來。

如果已知目前區塊的起點是 iii，那與 n/in/in/i 擁有相同商數的區塊終點 jjj 可以用以下公式求出：$$j = \lfloor \frac{n}{\lfloor \frac{n}{i} \rfloor} \rfloor$$

知道起點 iii 和終點 jjj 後，區塊的總和即：(j - i + 1) * (n / i)

這怎麼求的？

假設現正在處理某個特定的數字 iii（區塊的起點），且已知 nnn 除以 iii 的商數，則令商數為 kkk：$$k = \lfloor \frac{n}{i} \rfloor$$

目標是要找到一個最大的整數 jjj，使得 nnn 除以 jjj 的商數仍然等於 kkk，即要滿足以下條件，並且讓 jjj 盡可能大：$$\lfloor \frac{n}{j} \rfloor = k$$

根據 floor 函數 ⌊x⌋\lfloor x \rfloor⌊x⌋ 的定義，如果 ⌊x⌋=k\lfloor x \rfloor = k⌊x⌋=k，就表示 xxx 必定落在 kkk（含）與 k+1k + 1k+1（不含）之間。

因此，可將 ⌊nj⌋=k\lfloor \frac{n}{j} \rfloor = k⌊jn​⌋=k 轉換成不等式：$$k \le \frac{n}{j} < k + 1$$

接下來透過代數運算，把 jjj 獨立出來，首先要將整個不等式取倒數（要注意因為 k,n,jk, n, jk,n,j 都是正整數，取倒數時不等號方向會反轉）：$$\frac{1}{k + 1} < \frac{j}{n} \le \frac{1}{k}$$

然後把不等式的每一項都乘上 nnn：$$\frac{n}{k + 1} < j \le \frac{n}{k}$$

看到這裡，再把以上這條對照 floor 函數的定義，最後你就會得出：$$j = \lfloor \frac{n}{k} \rfloor$$

然後再把 k 代回去得到這個公式的模樣：$$j = \lfloor \frac{n}{\lfloor \frac{n}{i} \rfloor} \rfloor$$

在 C++ 不用特地寫 floor() 因為 C++ 的 / 本身就在做這件事情了。

解題思路:

此為動態規劃（DP）演算法問題。

這題要在給定的長度 NNN 中，構造出滿足特定條件的二進位數字，並計算總共有幾種合法組合，不用真的把這些數字印出來，只需要計算數量即可，看到這裡就知道這題是 DP 了。

可用由左至右逐位填寫二進位數字的方式來思考，對於每個位置，僅可填入 0 或 1。

狀態定義：

要確定走到目前這一步時的狀態，首先要知道三件事：

  1. idx：目前填了多少個位元（從左邊數）。
  2. rem：目前構造出的數字，除以 KKK 的餘數是多少。
  3. ones：目前填了多少個 1。

在此建立一個三維陣列 dp[idx][rem][ones] 來存狀態，避免重複計算。

轉移式：

當於第 idx 位要往下填入數字時，有兩種選擇：

  1. 填 0：數字向左移一位，new_rem = (rem * 2 + 0) % K，1 的數量不變。
  2. 填 1：數字向左移一位並加一， new_rem = (rem * 2 + 1) % K。1 的數量++。

註：rem * 2 是因為現在在算二進位。
